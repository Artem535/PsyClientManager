== Testing

Tests live in `test/` and are wired from `test/CMakeLists.txt`.

=== Existing Tests

- `config_tests.cpp`
: basic construction test for `pcm::config::Config`.
- `database_tests.cpp`
: DB initialization test with temporary directory.

Framework: GoogleTest (`GTest`).

=== Execute

[source,bash]
----
ctest --test-dir build --output-on-failure
----

=== Current Coverage Shape

Current tests validate bootstrap-level behavior only:

- config object initialization,
- DB setup path and initial creation.

Not covered yet:

- UI/page interaction flows,
- timeline model/view behavior,
- event/client CRUD edge cases,
- model threading behavior (`QClientModel`).
