== Modules

=== `src/app`

- `application.*`
: app bootstrap and cross-page/business signal wiring.
- `main_window.*`
: root shell window, page registration, navigation, top custom widget host.

=== `src/config`

- `Config`
: YAML-backed app config.
- `DatabaseConfig::db_pth`
: DB directory path.
- `rfl_path.hpp`
: `Poco::Path` reflector for `rfl` serialization.

=== `src/database`

- `Database`
: CRUD for `Client` and `Event`, relation management (`EventClient`), conflict/day queries.
- `constants.hpp`
: SQL schema + seed statuses + demo data.
- `schema.hpp`
: DTO structs (`ObxClient`, `ObxEvent`, etc.) constructed from DuckDB chunks.
- `db_utils.hpp`
: value conversion helpers between DuckDB and C++ optional types.

=== `src/client_model`

- `QClientModel`
: `QAbstractListModel` for clients; async loading on worker thread.
- `QClientDelegate`
: custom card-like rendering and action hit-testing for list rows.

=== `src/pages/client_info_page`

- `ClientInfo`
: page with `QListView` bound to `QClientModel`.
- reacts to loading state and delegates "display" action.

=== `src/pages/event_info_page`

- `QEventInfoPage`
: calendar + timeline + event dialog orchestration.
- `QEventDetailsWidget`
: create/edit event form, validation, emits save/cancel/link signals.

=== `src/pages/detail_client_info_page`

- `QClientInfoCardPage`
: editable detailed client form.
- `QClient`
: Qt-friendly wrapper around `ObxClient` (conversion + computed age).

=== `src/event_view`

- `QTimelineModel`
: timeline data model for events of selected day.
- `QEventView`
: graphics view scene, timeline background/axis, item placement.
- `QEventItem`
: interactive event graphics object.

=== `src/timeline_widget`

- `QTimelineWidget`
: composition widget that connects `QEventView` with `QTimelineModel`.

=== `src/widgets`

- `TabButton`
: custom painted navigation button.
- shared UI constants.
